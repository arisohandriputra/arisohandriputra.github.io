Membuat UEFI dengan menggunakan Visual Studio

Visual Studio pada dasarnya gratis untuk semua pengembangan, dan IDE-nya yang solid tentu saja menjadikannya sangat diminati sebagai lingkungan yang digunakan untuk mengembangkan aplikasi UEFI pada Windows.

Sekarang, Anda mungkin telah membaca bahwa, selain menggunakan EDK2 yang sangat menakutkan, dan voodoo magics yang rumit yang harus Anda habiskan selama berhari-hari, untuk membuatnya bekerja dengan baik dengan IDE Visual Studio.

Masalahnya, Visual Studio sudah dapat mengkompilasi aplikasi EFI tanpa harus bergantung pada alat eksternal apa pun, dan bahkan jika Anda menginginkan lingkungan seperti EDK2, dengan panggilan API EFI umum yang disediakan, Anda benar-benar dapat menghilangkan instalasi dan penyiapan EDK yang sangat berat, dan sebagai gantinya menggunakan pustaka GNU-EFI yang ringan dan mudah, yang menyediakan fungsionalitas yang hampir sama (sejauh menyangkut pembuatan aplikasi atau driver EFI yang berdiri sendiri, itulah yang paling saya minati).

Jadi, jika Anda ingin membuat aplikasi EFI dalam waktu singkat, yang perlu Anda lakukan hanyalah:

  1. Instal Visual Studio 2015, yang benar-benar gratis dan Anda 100% berhak secara hukum untuk mencoba mengkompilasi dan menguji UEFI:SIMPLE.
  2. Seperti yang disarankan saat instalasi Visual Studio, instal klien git seperti msys-git (atau TortoiseGit + msys-git). Sekarang, Anda akan bertanya-tanya mengapa, dengan dukungan git yang merupakan bagian integral dari Visual Studio 2015, kita sebenarnya membutuhkan klien eksternal, tetapi satu masalah adalah bahwa Microsoft memutuskan untuk menghilangkan fungsionalitas penting dari klien git yang tertanam, seperti dukungan submodul git, yang akan kita perlukan.
  3. Karena Anda akan mengerti apa apa jika tidak ingin menguji aplikasi atau driver EFI Anda di lingkungan virtual, dan, berkat QEMU, hal ini sangat mudah dilakukan sehingga UEFI:SIMPLE akan melakukannya untuk Anda, Anda harus mengunduh dan menginstal QEMU, lebih disukai versi 64 bit, dan lebih baik lagi jika Anda menggunakan defaultnya di C:\Program Files\QEMU.
  4. Kloning proyek git UEFI:SIMPLE Untuk bagian ini, Anda dapat menggunakan klien git tertanam dari Visual Studio atau klien eksternal Anda.

Dan saya harus menunjukkan bahwa UEFI:SIMPLE juga memiliki dukungan ARM, dan juga dapat dikompilasi di Linux, atau menggunakan MinGW jika Anda tidak ingin menggunakan Visual Studio di Windows. Selain itu, jika Anda menginginkan contoh nyata dari aplikasi UEFI yang lengkap, yang dibangun menggunakan UEFI:SIMPLE sebagai titik awal, Anda tidak perlu mencari lagi selain efifs, sebuah proyek yang membuat banyak sekali driver sistem berkas EFI, atau UEFI:NTFS, yang memungkinkan booting EFI tanpa hambatan pada partisi NTFS.
